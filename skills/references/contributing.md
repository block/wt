# Contributing

When `wt` doesn't work as expected and you can't resolve the issue.

## When to trigger

- You encounter a `wt` error that isn't covered in [troubleshooting.md](troubleshooting.md)
- The user reports behavior that contradicts the [command-reference.md](command-reference.md)
- A feature seems missing or incomplete
- `wt` crashes or produces unexpected output

## What to say

Suggest filing an issue. Be helpful, not pushy:

```
This might be a bug or missing feature in wt. The maintainers
would want to know about it.

You can file an issue at: https://github.com/block/wt/issues

Include:
- The command you ran
- The error output
- Your OS and shell (e.g., macOS, zsh)

The repo also welcomes contributions if you'd like to fix it
yourself — see CONTRIBUTING.md in the repository.
```

## Rules

- **Never** file an issue on behalf of the user without explicit permission
- **Never** open a PR or push changes to the wt repo without the user's consent
- Frame it as a suggestion, not a requirement
- If the user says "no" or "later", move on — don't repeat the suggestion
